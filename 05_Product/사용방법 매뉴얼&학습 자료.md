# CLI Tutor 사용 방법 매뉴얼 & 학습 자료

본 문서는 CLI Tutor의 효율적인 사용 방법과 주요 기능을 설명합니다.

---

## 1. 화면 구성 설명

CLI Tutor는 총 4개의 주요 영역으로 나뉩니다.

1.  **📋 세션 목록 (좌측)**
    *   현재 활성화된 학습 세션을 관리합니다. 
    *   `[+ 새 세션]`을 클릭하거나 마우스 휠로 세션 간 이동이 가능합니다 (향후 자동 저장 지원).
2.  **💻 메인 터미널 (중앙)**
    *   명령어가 직접 실행되는 공간입니다. 
    *   창 크기에 따라 텍스트가 잘릴 수 있으므로, 충분한 너비를 확보한 상태에서 사용하세요.
3.  **💡 명령어 해설 (우측 상단)**
    *   시스템이 방금 실행된 명령의 의미와 결과를 분석해 줍니다.
    *   창 크기가 충분하지 않으면 해설 내용이 생략될 수 있으니, 우측 패널이 넉넉하게 보이도록 조정하세요.
4.  **🤖 에이전트 가이드 (우측 하단)**
    *   특정 목표(`/goal`)를 입력했을 때, 단계별 학습 시나리오를 제시합니다.

---

## 2. 세션 사용 및 관리 방법

*   **독립적인 작업 환경**: 각 세션은 독립된 히스토리를 가집니다. 프로젝트 A용 세션과 프로젝트 B용 세션을 나누어 쓸 수 있습니다.
*   **세션 이동**: 좌측 리스트에서 원하는 세션을 **마우스로 선택**하거나, 위젯 포커스(`Tab`)를 세션 리스트로 옮긴 뒤 방향키로 이동할 수 있습니다.
*   **초기화 중 아님**: 현재 버전에서는 앱 종료 시 세션 데이터가 완전히 저장되지 않을 수 있습니다. 중요한 실습 내역은 로그 파일을 통해 확인하세요.

---

## 3. 기본 조작 방법

### ⌨️ 주요 단축키
터미널 환경에 따라 일부 단축키가 작동하지 않을 경우 대안키를 사용하세요.

> [!CAUTION]
> **준수 사항: `Ctrl + C` 주의**
> 윈도우 터미널에서 `Ctrl + C`는 프로그램 강제 종료 명령어입니다. 
> 텍스트를 복사하려다 실수로 프로그램을 종료할 수 있으니 주의하세요!
> 
> *   **복사**: 마우스로 드래그 후 (터미널 설정에 따라) `Enter` 또는 `마우스 우클릭`.
> *   **붙여넣기**: `Ctrl + V` 또는 `마우스 우클릭`.

| 기능 | 기본 단축키 | 대안 단축키 |
| :--- | :--- | :--- |
| **입력창 포커스** | `Ctrl + G` | (입력창 클릭) |
| **터미널 지우기** | `Ctrl + L` | - |
| **프로그램 종료** | `Q` | `Ctrl + Q` |
| **모드/위젯 전환** | `Tab` | `Ctrl + T` |

### ⚠️ 주의사항 및 팁
- **대화형 셸 금지**: 입력창에 `powershell`, `cmd`, `python` 등 대화형 모드로 진입하는 명령어를 직접 치지 마세요. 앱이 응답하지 않을 수 있습니다. (향후 지원 예정)
- **화면 크기**: TUI 특성상 창이 너무 작으면 내용이 잘릴 수 있습니다. 터미널 창을 최소 100x30 이상으로 키워 사용하시기를 권장합니다. 잘릴 경우 창을 키우면 자동으로 다시 배치됩니다.
- **복사/붙여넣기**: 마우스 우클릭을 적극 활용하세요. `Ctrl+C`는 종료 버튼입니다!

1.  **일반 실행**: 그냥 명령어를 입력하고 `Enter`를 누르세요. (예: `ls -al`)
2.  **셸 전환**: `/shell` 뒤에 원하는 셸 이름을 입력하세요. (예: `/shell powershell` 또는 `/shell cmd`)
    *   이제부터 입력하는 모든 명령어는 해당 셸에서 실행됩니다. 윈도우에서 `ls`를 쓰고 싶다면 파워쉘로 전환해 보세요!
3.  **목표 설정**: `/goal` 또는 `/g` 뒤에 목표를 입력하세요. (예: `/goal git 초기 설정해줘`)
4.  **질문/설명**: `/explain` 또는 `/e` 뒤에 궁금한 점을 적으세요. (예: `/explain chmod 755가 뭐야?`)

## 3. 학습 가이드 (예시 시나리오)

### 초보자를 위한 Git 실습
1.  **목표 입력**: `/goal git 저장소 만들고 첫 커밋하기` 입력.
2.  **가이드 확인**: 우측 하단 **에이전트 가이드**에 표시된 1단계 명령어를 확인합니다.
3.  **명령 실행**: 중앙 터미널에 `git init` 입력 후 실행.
4.  **해설 확인**: 우측 상단 **명령어 해설**에서 `git init`이 무엇인지, 성공했는지 확인합니다.
5.  **반복**: 가이드의 다음 단계를 따라하며 학습을 이어갑니다.

## 4. 트러블슈팅 (Q&A)

**Q: 엔터를 쳤는데 반응이 없어요.**
A: 일부 터미널 환경에서는 `Enter` 키가 텍스트(`\n`)로 삽입될 수 있습니다. 이 경우 프로그램을 실행 중인 터미널 설정을 확인하거나, `Ctrl+J` 또는 `Ctrl+M`을 시도해 보세요.

**Q: 화면이 깨져 보여요.**
A: 터미널 창의 크기를 충분히 키운 후 프로그램을 재시작(`python -m cli_tutor`) 하세요. (최소 150x40 권장)
