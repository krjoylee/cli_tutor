# 05. CLI Tutor Project Philosophy (학습의 철학)

본 문서는 `CLI_Tutor` 프로젝트의 존재 이유와 철학적 기반을 정의합니다. 초기 구상했던 특정 LLM(Gemini) 종속성을 탈피하고, 보다 가볍고 범용적인(General) AI 학습 도구로서의 철학을 확립합니다.

---

## 1. 서비스의 본질 (Essence)

> **"터미널은 더 이상 두렵지 않다. 가장 가볍고 친절한 나만의 CLI 전속 튜터."**

**CLI Tutor** — "내 터미널 속의 마스터."

명령줄 인터페이스(CLI)는 강력하지만, 초보자에게는 검은 화면과 흰 글씨라는 시각적 압박감과 높은 진입 장벽을 제공합니다.
이 서비스는 사용자가 터미널 환경을 두려워하지 않고, **직접 입력하고 깨달음을 얻는 학습의 모멘트(Momentum Point)**를 제공합니다. 명령어에 대한 단순한 검색을 넘어, 내 시스템 환경에 맞춘 정확한 가이드를 통해 사용자 스스로 성장할 수 있도록 돕습니다.

---

## 2. 인간의 본능과 작동 원리 (Human Operating System)

### 2.1 미지의 환경에 대한 두려움 (Command Line Phobia) 해소
CLI 환경에서는 마우스 클릭과 같은 시각적 유도선이 없습니다. 잘못된 명령어가 시스템을 망가뜨릴 수 있다는 무의식적인 공포가 존재합니다.
**서비스 적용:**
- 사용자가 명령어를 입력했을 때, 그것이 시스템에 어떤 영향을 미치는지 사전에(또는 실행 직후에) 친절히 해설하여 불안감을 제거합니다.

### 2.2 즉각적인 피드백과 학습 동기 부여 (Feedback Loop)
인간은 자신의 행동에 대한 명확하고 즉각적인 결과를 시각적으로 확인할 때 가장 빠르게 학습합니다.
**서비스 적용:**
- 콘솔 출력결과나 에러 코드를 그대로 내뱉는 것이 아니라, **Explanation Panel**을 통해 성공/실패 여부를 사람의 언어로 번역해 줍니다. "무엇이 잘못되었는지"와 "어떻게 고칠 수 있는지"를 즉시 제공하여 좌절을 방지합니다.

### 2.3 목표 지향적 몰입 (Goal-Oriented Flow)
"Git 설정은 어떻게 해?" 와 같은 목표가 주어졌을 때, 방대한 문서를 던져주기보다 스텝 바이 스텝으로 진행할 때 몰입도가 높아집니다.
**서비스 적용:**
- **Agent Panel**을 통해 사용자가 달성하고자 하는 목표를 시나리오(Step 1, Step 2...) 단위로 쪼개어 가이드합니다.

---

## 3. 핵심 철학 명제 (Core Thesis)

### 명제 1: 가벼워야 한다. (무료, 로컬 지향)
> 무거운 웹 API나 유료 구독 모델보다, CLI 특성에 맞게 가장 가볍게 동작해야 합니다. Groq와 같은 초고속/무료 모델을 1순위로 채택하여 진입 장벽을 완전히 낮춥니다.

### 명제 2: 설명은 사람의 언어로 직관적이어야 한다.
> "Exit code 1" 이라는 결과보다, "접근 권한이 부족합니다. sudo를 붙여서 다시 실행해보세요." 라는 직관적 해설이 필요합니다. 터미널의 차가움을 AI의 온기로 중화시킵니다.

### 명제 3: 범용성 (General Concept)
> 특정 서비스(Gemini, OpenAI 등)에 묶이지 않습니다. 사용자의 취향이나 상황에 따라 가장 적합한 LLM(Groq, Perplexity 등)으로 유연하게 교체할 수 있어야 하며, 구조는 확장에 열려 있어야 합니다.

### 명제 4: 정답을 지시하지 않고, 방향을 제안한다.
> 코드를 자동으로 완성해 주고 끝내버리는 것이 아니라, 다음으로 할 수 있는 명령어를 1~2개 '제안'하여 사용자가 직접 타이핑하고 실행하는 행동(Action)을 유도합니다.

---

## 4. 서비스 지향점 (Vision Statement)

| 구분 | 내용 |
| :--- | :--- |
| **미션** | CLI 환경의 공포를 없애고, 누구나 쉽게 시스템과 대화하는 방법을 학습하게 한다. |
| **비전** | 전 세계 개발자와 시스템 관리자의 첫 번째 터미널 동반자 |
| **핵심 가치** | 가벼움(Lightweight), 친절함(Friendly), 범용성(Generality), 자립심(Self-Reliance) |
