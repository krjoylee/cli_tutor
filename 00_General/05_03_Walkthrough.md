# 05. 프로젝트 웍스루 (Walkthrough)

본 문서는 `CLI Tutor Project`의 작업 흐름, 운영 표준, 협업 규칙을 설명합니다.
본 프로젝트는 독립 배포가 가능하며, 외부 협업자도 이 문서만으로 운영 규칙을 이해할 수 있습니다.

## 1. 작업 워크플로우 (Workflow)

1.  **아이디어 포착**: `01_Expansion/` 폴더 내에 자유로운 아이디어 스케치(Card) 작성.
2.  **구조화**: `05_02_Template.md`를 사용하여 `entries/`에 정식 문서 생성.
3.  **검토 및 정제**: 설계의 논리적 결함 검토 및 `02_Refinement/` 정리.
4.  **기각 보관**: 채택되지 않은 아이디어는 `03_Discard/`에 사유와 함께 보관. (삭제하지 않음)
5.  **개발 착수**: 확정된 설계를 바탕으로 프로토타입 코딩 및 구현 진행.

## 2. 폴더 구조 가이드

- **00_General/**: 프로젝트 관리 및 거버넌스 문서 (본 파일 포함)
- **01_Expansion/**: 아이디어 확장, 브레인스토밍, 기능 카드 (Card)
- **02_Refinement/**: 구체화된 기획안, 정제된 요구사항, 알고리즘 설계
- **03_Discard/**: 채택되지 않은 아이디어 보관소 (삭제하지 않음)
- *(샘플 프로토타입은 별도 프로젝트로 분리 관리됨)*
- **entries/**: 최종 확정된 설계 문서, 산출물, 코드 스니펫

## 3. 운영 표준 (Operating Standards)

본 프로젝트에서 준수해야 할 핵심 운영 규칙입니다.

### 3.1 문서 작성 규칙
| 규칙 | 내용 |
| :--- | :--- |
| **인코딩** | 모든 문서는 반드시 `UTF-8`로 작성하며, 중간에 임의 변경을 금지합니다. |
| **파일 이름** | `NN_Name.md` 또는 `NN_MM_Name.md` 형식으로 넘버링합니다. |
| **제목 수준** | H1(`#`)부터 순차적으로 사용하며, 갑자기 건너뛰지 않습니다. |
| **빈 줄** | 블록 요소(제목, 표, 코드블록) 사이에 반드시 빈 줄을 삽입합니다. |

### 3.2 Git 운영 규칙
| 규칙 | 내용 |
| :--- | :--- |
| **기본 브랜치** | `main` 브랜치를 사용합니다. (`master` 사용 금지) |
| **커밋 메시지 언어** | 반드시 **한글**로 작성합니다. |
| **커밋 메시지 형식** | `타입: 설명` (예: `기능: 시나리오 생성 모듈 추가`) |
| **타입 분류** | `기능`, `수정`, `문서`, `리팩토링`, `설정` |
| **동기화** | 중요 마일스톤 달성 시 반드시 원격 저장소에 푸시합니다. |

### 3.3 넘버링 규칙 (Numbering Procedure)
- 프로젝트 내 파일: `NN_MM_Name.md` 형식 (2단 시퀀스)
- 폴더 깊이가 깊어져도 파일명은 `NN_MM_` 형식을 유지합니다.
  - 예: `01_03_01` ❌ → `01_05` ✅
- `00_` 접두사: 공통(Common)/기반(Foundation) 폴더를 의미하는 특수 접두사
