## 1. 개요 (Background)
CLI Tutor의 주된 잠재 고객(초심자, 주니어 개발자, 비전공 시스템 관리자 등)이 겪는 페인 포인트(Pain Point)를 해결하기 위한 핵심 사용 사례를 정의합니다. TUI 분할 환경(터미널 + 에이전트 + 설명)에 맞춘 특화된 사용자 흐름을 구축합니다.

## 2. 상세 설계 (Detailed Design)

### 2.1 Use Case 1: 목표 기반의 능동적 학습 가이드 (Goal-Oriented Tutor)
> 복잡한 도구나 셸 스크립팅, Git 등의 사용법을 몰라 어떻게 시작할지 막막할 때.

- **상황:** 사용자는 "Git으로 처음 리포지토리를 만들고 원격에 푸시하고 싶어"와 같은 추상적인 목표를 Agent Panel에 입력합니다.
- **동작:**
  1. 목표를 LLM(Groq/Perplexity)에 전달하여 사용자의 OS(Windows/Linux) 환경에 맞는 3~5단계 최적 시나리오를 생성.
  2. 에이전트 패널에 각 단계의 명령어와 간단한 설명 표시.
  3. 사용자는 터미널 패널에서 제시된 명령어를 한 줄씩 따라 입력하며 능동적으로 학습.

### 2.2 Use Case 2: 터미널 출력 및 에러 즉각 해설 (Real-time Explanation & Troubleshooting)
> 명령어를 쳤는데 "Permission denied" 또는 "command not found" 등의 빨간/일반 에러 로그가 쏟아질 때.

- **상황:** `npm install` 이나 `docker build` 등을 실행하다가 원인을 알 수 없는 롱테일 에러를 맞닥뜨림.
- **동작:**
  1. 사용자가 메인 터미널에서 명령을 실행하면, 시스템이 해당 명령어와 출력결과, 종료 코드(exit code)를 수집.
  2. 백그라운드에서 LLM에 해당 로그 분석을 요청 (명령어 + 출력값 + 환경정보).
  3. 우측 상단의 Explanation Panel에 "발생 원인"과 "가장 가능성 높은 해결책(다음 명령어 1~2개 제안)"을 사람의 언어(한국어)로 즉시 렌더링.
  4. 구글을 켤 필요 없이 즉각적인 문제 해결 (Context Switching 최소화).

### 2.3 Use Case 3: 새로운 명령어/플래그 탐색 (Discover & Try)
> 특정 기능을 위해 명령어가 존재하는지, 어떤 옵션(-f, -a 등)을 줘야 하는지 찾고 싶을 때.

- **상황:** "이 폴더 안에 있는 100메가 넘는 파일만 찾아줘" 등.
- **동작:**
  1. 에이전트 패널에 단순 자연어로 질문.
  2. 에이전트가 "OS 맞춤형 명령어"(`find . -type f -size +100M`)를 제안.
  3. 사용자는 터미널에 입력, 동작 여부를 설명 패널을 통해 크로스 체크.

## 3. 기대 효과 및 고려 사항 (Impact & Considerations)
- **도입 시 장점:** 에러 공포증 해소. 즉각적이고 개인화된 과외 교사를 두는 느낌.
- **잠재적 문제점:** 터미널 출력(stdout, stderr)이 구동 중에 연속적으로 스트리밍되는 명령어(예: 무한 루프나 서버 기동)는 어떻게 캡처하고 분석할지 TUI 단에서의 PTY(Pseudo-terminal) 제어 기술 추가 연구가 필요함.
