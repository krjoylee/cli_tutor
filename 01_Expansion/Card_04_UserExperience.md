## 1. 개요 (Background)
텍스트 기반 인터페이스(TUI)는 그 특성상 정보 밀도가 높으면 시각적으로 피로해집니다. CLI Tutor는 명확한 공간 분할과 풍부한 텍스트 스타일링(Rich 라이브러리 활용)을 통해 "흑백 터미널"을 "살아 숨쉬는 대시보드"로 진화시킵니다.

## 2. 상세 설계 (Detailed Design)

### 2.1 마법사 경험 (Onboarding Wizard)
사용자가 `cli-tutor` 명령어를 최초로 실행했을 때, 복잡한 README 파일을 던져주는 대신 인터랙티브한 대화형 Setup 마법사가 동작합니다.
- 내 시스템 감지 상태 (Windows인지 Linux인지) 시각적 렌더링.
- 추천 LLM 1순위(보통 무료인 Groq)를 기본값으로 제시.
- API 발급 링크 제공 및 프롬프트 상에서 Key 즉각 입력.

### 2.2 메인 인터페이스 구조 (4-Panel Grid)

| 영역 | 패널명 | 주요 역할과 UX적 시각 표현 |
| :--- | :--- | :--- |
| **좌측 막대** | **Session Panel** | 과거에 수행했던 질문/작업 기록 (세션 분리용). 색상을 통한 활성 세션 구분. |
| **중앙 메인** | **Terminal Panel** | 실제 명령어를 타이핑하고, 셸 출력이 보이는 핵심 구역. (`녹색 빛` 강조 테두리로 집중) |
| **우측 상단** | **Explanation Panel** | 입력한 명령어에 대한 해설. 실패 시 `붉은 계열 테마` 배경이나 텍스트로 에러 강도 표시. 성공 시 안심감을 주는 차분한 톤 다운 유지. |
| **우측 하단** | **Agent Panel** | 사용자 목표 입력 창구 및, 시나리오 스텝 표시. "무엇을 해야 할지" 길을 밝혀주는 네온 포인트 컬러 활용. |

### 2.3 UX Flow 시퀀스

1. 사용자가 Agent Panel을 통해 "나의 목표" 텍스트 입력 후 엔터.
2. 0.5초 이내에 Agent Panel의 내용이 "Step 1. 이 명령어(ex. `git init`)를 입력 해보세요"로 전환.
3. 사용자가 Terminal Panel에 의도적으로 오타(`git initt`)를 친 후 엔터.
4. Terminal이 "unknown subcommand" 출력.
5. 곧이어 1초 이내에 우측 상단 Explanation Panel 상단에 "❌명령어 오타입니다. initt 대신 init을 사용하세요" 렌더링.
6. 안도감을 얻은 사용자가 명령어 수정 후 재실행. 성공. Explanation 패널이 "✅초기화 성공!" 등으로 변화.

## 3. 기대 효과 및 고려 사항 (Impact & Considerations)
- **정보 과부하 방지:** 터미널에서는 모든 텍스트가 위에서 아래로 흐릅니다. 튜터 앱에서는 고정된 패널 위치 덕분에 "명령어", "해설", "시나리오"의 시선이 분산되지 않고 구획 안에 깔끔히 갇혀 학습 효율이 올라갑니다.
- **과제:** Textual의 입력기(Input)와 셸 상호작용의 자연스러운 동기화를 UI 상에서 끊김없이 구현하는 UX적 디테일(로딩 인디케이터 처리 등) 필요.
