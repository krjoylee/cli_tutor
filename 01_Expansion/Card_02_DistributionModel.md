## 1. 개요 (Background)
프로토타입이 `Gemini` API 종속성에서 벗어나 `Groq`, `Perplexity` 등의 다중/무료 모델을 지원하도록 변모한 근본적 이유를 확장합니다. 이는 수익화(Revenue)를 직접 목표로 하기보다 **"가장 대중적이고 영향력 있는 (무료) 학습 생태계 구축"**을 통한 플랫폼 파워 획득에 중점을 둡니다.

## 2. 상세 설계 (Detailed Design)

### 2.1 제품 포지셔닝 (Product Positioning)
- **무료 우선 (Free-First):** 과금 없이 누구나 사용할 수 있게 하여 초기 진입 장벽을 완전히 0으로 수렴. (`Groq`의 무료 API 티어를 적극 활용)
- **로컬 구동, 개인화 프라이버시 제한:** 개인의 로컬 폴더(`~/.cli-tutor`)에 설정 파일과 세션을 저장, 외부 의존도 최소화.

### 2.2 Growth Phase Model (성장 3단계)

| Phase | 단계명 | 핵심 목표 | 배포 / 실행 전략 |
| :--- | :--- | :--- | :--- |
| **Phase 1** | **오픈소스 / 개발자 유입** | 패키지 인지도 확보 | `pip install cli-tutor` 처럼 파이썬 패키지로 릴리스. Github Star 확보, 커뮤니티(레딧, 개발자 카페) 공유 집중. |
| **Phase 2** | **프리미엄 튜토리얼 마켓페이퍼** | 생태계 구축 집중 | 단순 명령어 분석을 넘어, "Node.js 세팅 튜토리얼 팩", "Kubernetes 정복 팩" 형태의 JSON 시나리오 템플릿(세션 데이터) 공유 문화 형성 유도 |
| **Phase 3** | **엔터프라이즈 / B2B 확장** | 보안 및 맞춤형 플러그인 | 기업망 내부(온프레미스)에 존재하는 사내 Local LLM(ex. Ollama, vLLM) 연동 기능 오픈. 기업 고유의 사내 망 복잡한 명령어 도구 학습 튜터로 세일즈. |

## 3. 기대 효과 및 고려 사항 (Impact & Considerations)
- **기대 효과:** 초기 비용 압박이 없기 때문에 입소문을 타고 빠른 사용자 확보 가능성. 프로모팅이 쉽고, 사용자들의 환경 구성 자유도(API 키 본인 지참)로 인해 서버 호스팅 유지 비용 제로.
- **잠재적 문제점:** API 의존성. (예: Groq가 무료 티어를 축소하는 정책 변화 시 대안 필요). 이는 로컬 LLM 연동 아키텍처 지원으로 극복 모색.
